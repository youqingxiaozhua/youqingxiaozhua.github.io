产品概念


主机

虚拟化

虚拟化是一种将计算机实体资源进行整合的资源管理技术。实体计算机的资源如： CPU 、内存、硬盘存储、网络经过虚拟化技术，突破了原有的地域或者物理配置的限制，用户可以更好的使用这些资源。

云主机

云主机是基于服务器虚拟化技术的一种主机服务。通过虚拟化技术，整合高性能服务器资源，用户可以根据业务需求自 由地配置所需的云主机的 CPU 、内存、硬盘及网络资源，节省生产成本。基于 CloudIn 提供的镜像创建的可弹性伸缩的计算服务，您可以自定义主机配置，并随时动态扩容，快速构建更稳定高效的应用，更专注于自身核心业务。

主机实例

用户在云上所创建出的云主机就是一个主机实例。

系统镜像

系统镜像为主机实例提供一个初始的运行环境。目前 CloudIn 为您提供了 Linux 操作系统镜像以及 Windows 操作系统镜像，基于提供给您的系统镜像，您可以创建任意数量的云主机。

CloudIn支持您通过主机备份创建主机，根据备份创建出的主机，镜像类型、配置、数据、登录方式都与原主机备份时一致。

存储

云存储

CloudIn 基于下一代分布式存储系统，作为云主机、硬盘、镜像服务的统一存储平台。

硬盘

在 CloudIn 中，硬盘为主机提供了独立于主机的生命周期而存在的块级存储。

硬盘可以被连接到任意运行的主机上，也可以自由的解除连接。底层支持镜像复制、数据快照的能力，用户可以自定义硬盘大小，理论支持 TB 级， IO 吞吐带宽高达上百 Mb/s，硬盘的生命周期独立于所挂载云主机，支持全天候自动备份。

存储模式

CloudIn为您提供两种存储模式：存储型和效率型。

 

子网

网络是主机对外进行通信的媒介，您可以对创建的云主机进行网络配置，实现理想的网络功能。 CloudIn提供了两种网络方式：公网子网、内网子网。

公网子网

公网子网是用户独享的开放的网络，可以通过关联路由器与其他公网子网连接，亦可以通过路由器的网关功能连接到 Internet ，加入公网子网的主机能够访问其他关联同一路由器的公网子网的主机，并且，主机绑定公网IP 后，能够访问 Internet 。

内网子网

内网子网是用户独享的封闭的网络，内网子网没有网关功能，不能关联路由器，加入其中的主机仅能与同一内网子网的主机通信，具有绝对的隔离和安全性，类似于将主机接入到一台没有接入任何路由设备的交换机。

基础网络

基础网络是一个由 CloudIn 维护的默认公网子网，无需用户做任何配置与管理，即可直接使用， 但其只能通过依靠安全组来保障安全。基础网络的优点是简单易用，无需用户做任何的配置与管理，即可连接使用。用户也可自主创建子网并管理， 各子网之间是相互隔离的，以满足用户的安全需求。

 

路由器

路由器是在三层网络中进行数据转发的媒介。

CloudIn 为您提供了路由器进行数据转发，完成网络拓扑。

 

公网IP

公网IP地址是在互联网上合法的静态IP 地址。

在 CloudIn 系统中，公网IP 地址也是您的账户资源的一部分，您可以将申请到的公网IP 地址分配到任意的主机上，并随时可以解绑、再分配到其它资源。

 

备份

云备份是对指定时间点的数据进行可用拷贝，将系统数据加以复制备份，当设备发生故障或者文件遭到损坏时，能够及时有效的恢复数据，完成数据的快速回滚。

CloudIn 为您提供了主机备份和硬盘备份服务。

主机备份

CloudIn在您创建主机时为您提供了特定容量大小的系统盘，主机备份将对您的系统盘内的数据进行复制、保存。在必要的时候，您可以使用主机备份进行数据恢复或者创建一个具有相同数据的主机。

硬盘备份

您可能在系统盘之外需要额外的硬盘空间进行存储，通过 CloudIn 硬盘备份，可以对您的这部分数据进行复制、保存。在必要的时候，您可以通过硬盘备份进行数据恢复或者创建一个具有相同数据的硬盘。

 

镜像

系统镜像为主机实例提供一个初始的运行环境。

CloudIn支持您通过主机备份创建主机，根据备份创建出的主机，镜像类型、配置、数据、登录方式都与原主机备份时一致。

目前 CloudIn 为您提供了 Linux 操作系统镜像以及 Windows 操作系统镜像，基于提供给您的系统镜像，您可以创建任意数量的云主机。

根据不同类型的操作系统镜像， CloudIn 为您提供了不同大小的系统盘。

Linux的系统盘为20GB；Windows的系统盘为40GB，一旦创建则不可修改。
